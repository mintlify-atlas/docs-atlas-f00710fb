This page provides a comprehensive reference for all command-line options available in Quick Test CLI.

## Global Commands

Quick Test CLI provides several subcommands for different testing scenarios:

- `stress` - Verify code execution time and memory usage
- `cmp` - Compare target file with correct solution
- `check` - Validate answers using a custom checker script
- `output` - Run test cases and generate output files
- `setup` - Configure Quick Test settings
- `example` - Show usage examples for subcommands

---

## Stress Command

Verify that the code execution time does not exceed what is allowed.

### Usage

```bash
quicktest stress --target-file <FILE> [OPTIONS]
```

### Required Options

<ParamField path="--target-file" type="path" required>
  The target file to test. This is the solution you want to verify.
  
  **Short flag:** `-t`
  
  **Example:** `--target-file solution.cpp`
</ParamField>

### Optional Parameters

<ParamField path="--gen-file" type="path" default="">
  Generator file that produces random test cases.
  
  **Short flag:** `-g`
  
  **Example:** `--gen-file generator.cpp`
</ParamField>

<ParamField path="--timeout" type="u32" default="2000">
  Maximum execution time allowed in milliseconds.
  
  **Alias:** `--tout`
  
  **Example:** `--timeout 3000`
</ParamField>

<ParamField path="--memory-limit" type="u64" default="1000000000">
  Maximum memory allowed in bytes (default: 1GB).
  
  **Alias:** `--ml`
  
  **Example:** `--memory-limit 512000000`
</ParamField>

<ParamField path="--test-cases" type="u32" default="1000">
  Number of test cases to run.
  
  **Alias:** `--tc`
  
  **Example:** `--test-cases 500`
</ParamField>

<ParamField path="--prefix" type="string" default="">
  Prefix for saved test case files.
  
  **Short flag:** `-p`
  
  **Example:** `--prefix my_test`
</ParamField>

### Behavior Flags

<ParamField path="--break_bad" type="boolean">
  Break execution if WA, TLE, or RTE status occurs.
  
  **Short flag:** `-b`
  
  **Alias:** `--break`
  
  <Warning>
  The `check` command uses `--break_bad` (underscore) unlike other commands which use `--break-bad` (hyphen).
  </Warning>
</ParamField>

<ParamField path="--save-bad" type="boolean">
  Save only test cases with WA, TLE, or RTE status.
  
  **Conflicts with:** `--save-all`
</ParamField>

<ParamField path="--save-all" type="boolean">
  Save all test cases regardless of status.
  
  **Conflicts with:** `--save-bad`
</ParamField>

### Run Filters

<ParamField path="--run-all" type="boolean">
  Run all saved test cases.
</ParamField>

<ParamField path="--run-ac" type="boolean">
  Run only test cases with Accepted status.
</ParamField>

<ParamField path="--run-wa" type="boolean">
  Run only test cases with Wrong Answer status.
</ParamField>

<ParamField path="--run-tle" type="boolean">
  Run only test cases with Time Limit Exceeded status.
</ParamField>

<ParamField path="--run-rte" type="boolean">
  Run only test cases with Runtime Error status.
</ParamField>

<ParamField path="--run-mle" type="boolean">
  Run only test cases with Memory Limit Exceeded status.
</ParamField>

## Setup Command

Configure Quick Test settings and language configurations.

### Config Subcommand

Update configuration settings.

#### Usage

```bash
quicktest setup config --label <LABEL> --value <VALUE>
```

<ParamField path="--label" type="string" required>
  Label of the configuration to update. Format: `Language.SETTING`
  
  **Short flag:** `-l`
  
  **Example:** `--label "cpp.PROGRAM"`
</ParamField>

<ParamField path="--value" type="string" required>
  Value to set for the configuration.
  
  **Short flag:** `-v`
  
  **Example:** `--value "g++"`
</ParamField>

#### Examples

```bash
# Configure C++ compiler
quicktest setup config --label="cpp.PROGRAM" --value="g++"

# Set C++ standard
quicktest setup config --label="cpp.STANDARD" --value="-std=c++17"

# Configure Python interpreter
quicktest setup config --label="python.PROGRAM" --value="python3"
```

### Reset Subcommand

Reset all settings to defaults.

#### Usage

```bash
quicktest setup reset
```

This removes the configuration file at `~/.quicktest/languages.config.json`.
